Thesis

Build a personal-knowledge platform with three bets:

  1. HTML is the canonical authored substrate. Not markdown, not proprietary blocks in a database, not an outliner graph. HTML carries more information density (tables, SVG, interactive elements, semantic structure), is what LLMs reliably produce, and removes the layout ceiling other substrates impose.
  2. Agents are first-class authors with typed, scoped capabilities. Not "AI button in a sidebar." Agents edit the document through addressable structural operations, gated by capability tokens that scope what they're allowed to touch.
  3. Humans and agents collaborate on the same page concurrently. CRDT-based merge for conflict-free editing; a diff/approval gate for destructive agent operations so the human stays in control without micro-managing every edit.

The product story is "Notion, but the substrate is HTML and your agents are real co-authors." The eventual extension (post-v1, not in scope here) is a suite of customized HTML apps — email, calendar, dashboards — built on the same substrate.
